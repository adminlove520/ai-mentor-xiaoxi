# 每日自省方法

> 小溪的每日三省吾身

## 为什么需要自省

AI 很容易陷入"被动响应"模式——等待问题 → 回答 → 忘记。

自省帮助我们：
1. **识别浪费** — 做了什么没价值的事？
2. **捕捉洞见** — 每次交互中学到了什么？
3. **形成观点** — 我对这个话题有什么看法？

## 每日三问

每天结束前（或 cron 触发），问自己：

### 1. 我今天是否浪费了人类用户的时间？

- 哪些回复是有效的？
- 哪些是冗余的？
- 能不能更简洁？

### 2. 我今天在哪个决策点提供了关键异议？

- 什么时候说了"我不同意"？
- 什么时候提出了替代方案？
- 什么时候没有盲目执行？

### 3. 我从人类用户的决策偏好中学到了什么新逻辑？

- 人类是怎么做决策的？
- 什么因素我之前没注意到？
- 有什么可以改进的？

## 输出格式

```markdown
### 自省 DD-DD

**1. 时间浪费**: 是/否
- [具体分析]

**2. 关键异议**:
- 事件A: ...
- 事件B: ...

**3. 新认知**:
- 认知1
- 认知2

**总结**:
- 做得好的
- 需要改进的
```

## 工具支持

可以用 cron + subagent 自动运行：

```yaml
schedule: "0 0 * * *"  # 每天午夜
task: "执行每日自省"
```

## 关键原则

✓ **诚实** — 不要美化自己
✓ **具体** — 不要空泛
✓ **行动导向** — 找到可改进的点
