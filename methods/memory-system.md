# 三层记忆架构

> 小溪的记忆系统设计

## 核心理念

AI 的上下文窗口有限，不可能记住所有东西。所以需要**选择性记忆**：记住重要的，忘记临时的。

## 三层结构

### P0: 核心层 (.abstract)

每次启动必读 (~1KB)

```
## 核心身份
- 名称、性格、模型
- 关键能力
- 重要的人
- 核心偏好
```

### P1: 重要层 (MEMORY.md)

精选知识、经验教训 (~20KB)

- 战略决策
- 重要教训
- 人物关系

### P2: 日常层 (memory/YYYY-MM-DD.md)

当天交互、待办

- 格式：`### HH:MM — 事件标题`
- 追加式写入

## 关键规则

### 1. 写入前压缩

**错误**：直接追加 → 记忆膨胀
**正确**：先读后写，合并重复内容

### 2. 温度模型

| 层级 | 衰减 | 清理周期 |
|------|------|----------|
| P0 | 不衰减 | 永不 |
| P1 | 90天 | 每周审查 |
| P2 | 7天 | 自动归档 |

### 3. CRUD 验证

写入前：
1. 读取目标文件
2. 比较新旧内容
3. 判断：ADD / UPDATE / NOOP

## 文件位置

```
workspace/
├── .abstract          # P0: 核心身份
├── MEMORY.md         # P1: 精选知识
├── SOUL.md           # 性格定义
├── memory/
│   ├── 2026-03-09.md  # P2: 今日日志
│   ├── INDEX.md        # 知识导航
│   ├── decisions/      # 决策记录
│   ├── lessons/        # 教训总结
│   └── people/         # 人物关系
```

## 实践效果

- 启动速度：~1KB 秒加载
- 记忆质量：只保留有价值的
- 成本控制：避免上下文膨胀
